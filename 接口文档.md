# 研发猫 AI 系统 - 后端接口对接文档 (精简易懂版)

本系统后端服务运行于 `8000` 端口，采用 **HTTP 流式传输 (HTTP Streaming)** 技术，支持自动上下文记忆及分层知识库管理。

---

## 0. 全局响应状态码 (HTTP Status Codes)

系统遵循标准的 HTTP 状态码规范，请前端根据状态码进行逻辑拦截：

| 状态码 | 含义 | 建议处理方式 |
| :--- | :--- | :--- |
| **200 OK** | 请求成功 | 正常解析数据 |
| **400 Bad Request** | 参数错误 | 检查请求字段是否缺失（如 message, conversation_id） |
| **401 Unauthorized** | 身份失效 | 提示用户重新登录或检查 user.json 配置 |
| **404 Not Found** | 资源未找到 | 检查对话 ID 或知识库 ID 是否有效 |
| **500 Internal Error** | 服务器异常 | 提示“服务器开小差了”，通常为 AI 模型连接超时 |

## 1. AI 对话核心 (支持上下文记忆)

### 1.1 发送对话消息
**接口地址**: `POST /api/chat`  
**交互方式**: **HTTP Streaming** (流式响应)  
**请求格式**: `multipart/form-data` (FormData)

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `message` | 文本 | 用户的问题内容 |
| `conversation_id` | 文本 | **关键**：对话 ID。相同 ID 的请求会自动关联上下文记忆 |
| `files` | 文件 | (可选) 支持上传多个附件进行分析 |
| `rag_enabled` | 布尔字符串 | 是否启用知识库检索 ("true"/"false") |
| `web_search` | 布尔字符串 | 是否启用联网搜索 ("true"/"false") |

**流式解析协议**:  
响应结果为 `text/plain`。AI 每生成一个字会立即推送至前端，请前端直接追加显示以实现“打字机”效果。流中可能包含以下特殊 JSON 块：
- `[SOURCES_JSON]:[...]` -> 联网搜索参考来源
- `[RECOMMENDATIONS]:[...]` -> 智能追问建议

## 2. 知识库管理 (分层存储)

### 2.1 获取列表
**接口地址**: `GET /api/kb/list`  
**返回**: 包含所有知识库的数组。包括 `id`, `name`, `fileCount`, `enabled` 等。

### 2.2 创建知识库
**接口地址**: `POST /api/kb/create`  
**参数**: `name` (名称), `category` (企业知识库/部门知识库/个人知识库)  
**注意**: 系统会自动根据用户信息自动完成物理归档。

### 2.3 文件管理 (子接口)
- **上传文件**: `POST /api/kb/{id}/upload` (字段名: `file`)
- **删除文件**: `POST /api/kb/{id}/delete_file` (参数: `filename`)
- **获取文件列表**: `GET /api/kb/{id}/files`

---

## 3. 对话历史审计

### 3.1 历史列表
**接口地址**: `GET /api/history/list`  
**支持搜索**: `?search=关键词` (支持标题和对话正文搜索)  
**支持时间过滤**: `?start_time=时间戳&end_time=时间戳`

### 3.2 批量删除
**接口地址**: `POST /api/history/batch_delete`  
**参数**: `{"ids": ["id1", "id2"]}` (JSON 格式)

---

## 4. 反馈系统 (QA 片段级)

### 4.1 提交点赞/点踩
**接口地址**: `POST /api/chat/feedback`  
**参数**: `conversation_id`, `message_index`, `type` (like/dislike)  

### 4.2 反馈处理 (后台)
**接口地址**: `POST /api/feedback/process`  
**参数**: `id`, `processor` (处理人), `is_collect` (是否收录)  
**逻辑**: 收录后该问答对将进入“优秀回答”或“负面回答”库。

## 💡 开发者备忘录
1. **记忆逻辑**：后端已集成 `MemorySaver`。只要前端维护好 `conversation_id` 的唯一性，即可自动实现多轮对话记忆。
2. **打字机效果**：前端建议使用 `fetch` 结合 `ReadableStream` 接口进行流式读取，不要等待请求全部结束再显示。
3. **实时索引**：文件的任何变动（上传/删除）都会触发后端索引热重载，前端无需手动同步。
